以下是简版全流程前端对接文档（面向 HarmonyOS 前端），覆盖从下单到确认收货的完整链路。你可以直接给前端按此实现。

**简版购茶流程对接文档（Mock 流程）**

**1. 目标**
在开发/联调阶段，无需支付网关与管理员后台，用户可自行完成：
`创建订单 → 模拟支付 → 模拟发货 → 确认收货`

---

**2. 前置条件**
- 后端环境：`settings.payment_mode = "mock"`  
- 所有请求需携带 `Authorization: Bearer <token>`  
- 创建订单必须带 `Idempotency-Key`

---

**3. API 流程**

**3.1 创建订单**

- `POST /api/v1/orders`
- Headers:
  - `Authorization: Bearer <token>`
  - `Idempotency-Key: <uuid 或业务唯一串>`
- Body:
```json
{
  "cartItemIds": ["ci1", "ci2"],
  "addressId": "addr1"
}
```
- Response:
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "orderNo": "20260310123456xxxxxx",
    "status": "PENDING_PAYMENT",
    "totalCent": 16800,
    "paymentRequired": true,
    "idempotent": false
  }
}
```

---

**3.2 模拟支付（无需签名）**
- `POST /api/v1/payments/mock/pay`
- Headers:
  - `Authorization: Bearer <token>`
- Body:
```json
{
  "orderNo": "20260310123456xxxxxx",
  "success": true
}
```
- Response:
```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "success": true,
    "idempotent": false,
    "status": "PAID"
  }
}
```

---

**3.3 模拟发货（无需管理员）**
- `POST /api/v1/orders/{orderNo}/mock-ship`
- Headers:
  - `Authorization: Bearer <token>`
- Response:
```json
{
  "code": 0,
  "message": "ok",
  "data": { "success": true }
}
```

---

**3.4 确认收货**
- `POST /api/v1/orders/{orderNo}/confirm-delivery`
- Headers:
  - `Authorization: Bearer <token>`
- Response:
```json
{
  "code": 0,
  "message": "ok",
  "data": { "success": true }
}
```

---

**4. 状态说明**
订单状态流转：
```
PENDING_PAYMENT → PAID → SHIPPED → COMPLETED
```

可用动作（接口返回 `canXXX`）：
- `canPay`：PENDING_PAYMENT
- `canCancel`：PENDING_PAYMENT
- `canConfirmDelivery`：SHIPPED

---

**5. 页面联动建议**
- 订单提交成功后进入“待支付”页
- 点击“立即支付”调用 `/payments/mock/pay`
- 支付成功后显示“待发货”
- 前端可直接调用 `/orders/{orderNo}/mock-ship` 模拟发货
- 发货后显示“待收货”
- 点击“确认收货”调用 `/orders/{orderNo}/confirm-delivery`

---

**6. 常见问题**
- 422/401：检查 `Authorization` 是否正确  
- 400 `mock mode disabled`：确认后端 `payment_mode=mock`  
- `Idempotency-Key is required`：创建订单必须带此 Header
